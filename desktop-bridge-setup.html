TabSyncBridge

The desktop bridge enables Chromium-family browser sync.

For Chromium workflows, TabSyncBridge uses a companion desktop bridge with the Chromium extension so browser tabs can participate in the private local-network sync loop.

View on the App Store Chrome Web Store companion

Why the bridge is needed

  • Native messaging Chromium-family browser workflows use native messaging through the desktop bridge.
  • Local relay The bridge helps relay local sync state between the browser extension and your linked devices.
  • Profile identity Browser profiles remain distinct so the same Chromium profile does not show its own tabs as remote tabs.
  • No cloud account The core workflow remains local to your own devices on the same private network.

When you need it

You need the desktop bridge for Chromium-family browser workflows. Safari workflows use the iOS app and Safari extension, while Chromium-family browsers use the Chromium extension together with the desktop bridge.

If native messaging is not detected, reinstall the desktop bridge and restart the browser.
